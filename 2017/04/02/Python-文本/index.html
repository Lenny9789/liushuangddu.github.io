文本 & 字符串

对Python 程序员来说,最显而易见的文本处理工具就是 string 类,不过除此之外,标准库中还提供了大量的其他工具,可以轻松地完成高级文本处理.
用 Python 2.0 之前的版本编写的老代码使用的是 string 模块的函数,而不是 string 对象的对象方法,对应这个模块中的每一个函数都有一个等价的方法,新的代码已经不再使用那些函数.
使用 Python 2.4 或之后的版本的程序可能会使用 string.Template 作为一个简便方法,除了具备 string 或 Unicode 类的特性,还可以对字符串实现参数化,与很多 web 框架定义的模板或者 Python package Index 提供的扩展模块相比,尽管 string.Template 没有那么多丰富的特性,但作为用户可修改的模板,即需要在静态文本中插入动态值,他确实很好的做到了二者兼顾.
textwrap 模块包括一些工具,可以从段落中抽取的文本进行格式化,比如限制输出的宽度,增加缩进,以及插入换行符从而能一致地自动换行.
除了对 string 对象支持的内置相等性和排序性比较之外,标准库还包括两个比较文版值有关的模板, re 提供了一个完整的正则表达式库,但是由于速度原因,这个库是由 C 语音实现的.正则表达式非常适合在比较大的数据集中查找子串,能够根据比固定字符串更为复杂的模式比较字符串,还可以完成一定程度的解析.
另一方面, difflib 则会根据添加,删除或修改的部分来计算不同文本序列之间的具体差别. difflib 中比较函数的输出可以用来为用户提供详细的反馈,之处两个输入中实现变化的地方,一个文档随时间有哪些改变等等.

string– 文本常量和模板

string 模块可以追溯到最早的 Python 版本,到了2.0版本,原先仅在这个模块中实现的很多函数则被移植为 str 和 Unicode 对象的方法.任然在遗留代码中使用这些函数,不过如今这些函数已经废弃,将在 Python3.0 中完全去除, string 模块保留了很多有用的常量和类,用来处理 string 和 Unicode 对象.

函数

还有两个函数未从 string 模块中移除: capwords() 和 maketrans().capwords() 的作用是将一个字符串中所有单词的首字母大写.

其结果等于先调用 split(), 这会将结果列表中各个单词的首字母大写,然后调用 join() 合并结果.

maketrans()函数将创建转换表,可以用来结合 translate() 方法将一组字符修改为另一组字符,这种做法比反复调用 replace() 更为高效.

模板与标准字符串拼接有一个重要区别,即模板不考虑参数类型,值会准换为字符串,再将字符串插入到结果中,这里没有提供格式化选项,例如没有办法控制使用几位有效数字来表示一个浮点数.
不过,这也有一个好处,通过使用 safe_substitute()方法,可以避免未能提供模板所需参数值时可能产生的异常.

由于 values 字典中没有对应的 missing 的值,因此 substitute() 会产生一个 KeyError. 不过 safe_substitute()不会抛出这个错误,他将捕获这个异常,并在文本中保留变量表达式.
