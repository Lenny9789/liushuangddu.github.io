总结一下五一两天假期所解决的 bug

在APP 打包上架 Store 的时候,Upload to App Store 时不出什么问题基本也就没什么问题,但是一旦出了问题,有可能会浪费很多时间,在这里总结一下常见的问题以及我是怎么做的.

ERROR-ITMS :90635

这个问题很简单,就是bitcode 的问题,但是网上的方法基本都是关掉bitcode, 其实没有必要,关于 bitcode 只需要保持你本身的项目支持 bitcode 的同时如果有导入其他的第三方库也使其支持 bitcode 就行了,对于日常的开发用到的框架基本上都是支持 bitcode 的;但是难免有的不支持,这个时候关掉 bitcode, 再把其他的开源框架关掉 bitcode 重编译一下再倒入就行,这个时候 bitcode 的问题基本可以解决,但是有时候 error 是一环扣一环,有可能这个 error 解决了又出了别的问题,一时找不到办法可能会认为刚才上一步是不是做错了什么,其实我们更应该保持一个平常心, bug 一个一个解决相信问题很快就能 pass.

ERROR-ITMS :90171

错误提示: “Invalid Bundle Structure Can’t Contain Standalone Executables”.

再引入第三方framework的时候大多数库的资源文件夹里面都会包含有一个 Info.plist 文件,这个文件是用来描述 framework 的配置情况. Info.plist 文件中有一个字段是” Excutable file”,也就是可执行文件,这个 framework 的可执行文件.按照常理来说的话每一个 framework 里面都应该有一个可执行文件,要不然这个库还有什么用,关于90171这个错误网上的说法就是直接删除” Excutable file”这个字段.其实这是错误的(有点脑子好不好0.0!).再删除这个字段后 framework 就丢失了可执行文件,找不到可执行文件自然就会报错” bundle has no excutable file”. 这个问题一般发生在闭源的 framework 上,也就是第三方公司提供的 framework, 开源的 framework 一般不会发生这个问题,因为可以自己修改.

其实解决这个错误很简单,至于本人产生这个问题确实因为粗心大意,本来在 Xcode 工程中导入 framework 一般是直接在” Build Phase” —> “Link Binary With Libraries”中直接添加就行了,而至于开源的 swift framework 则需要在”General” —> “Embedded Binaries”中添加,而我在” General “ —> “Embedded Binaries” 中添加了 BmobSDK.framework, 然后一系列的问题就由此产生,解决这一系列的问题就浪费了我两天美好的五一假期.

Bmob 的坑

在 bmob 官网上下载的SDK 是包含有 x86_64和 i386架构的,在电脑模拟器和真机都可以运行,但是上传到 App Store 就不行了,因为 .ipa 是不能包含 x86_64和 i386架构的,但是下载的 SDK 我们又不能改.

在这里只需要将 SDK 中的可执行文件提取出我们所需要的架构就行了

  • lipo -info 显示可执行文件所支持的架构
  • lipo “file” -thin armv7 -output “file2” 提取armv7 架构
  • lipo -create “file1” “file2” -output “file” 合并架构
